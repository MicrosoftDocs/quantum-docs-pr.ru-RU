# <a name="broombridge-specification-v01"></a>Спецификация брумбридже 0,1 #

Ключевые слова "обязательно", "не должны", "обязательно", "должно", "не", "должно", "не должно", "рекомендуется", "Май" и "необязательно" в этом документе должны интерпретироваться, как описано в [RFC 2119](https://tools.ietf.org/html/rfc2119).

Любая боковая панель с заголовками «Примечание», «информация» или «предупреждение» является информативной.

## <a name="introduction"></a>Общие сведения ##

Этот раздел является информативным.

Документы брумбридже предназначены для обмена экземплярами проблем моделирования в тактовой химия для обработки с помощью моделирования тактов и программирования цепочек инструментов.

## <a name="serialization"></a>Сериализация ##

Этот раздел является нормативным.

Документ Брумбридже должен быть сериализован как [документ YAML 1,2](http://yaml.org/spec/) , представляющий объект JSON, как описано в [RFC 4627](https://tools.ietf.org/html/rfc4627) , раздел 2,2.
Объект, сериализованный в YAML, должен иметь свойство `"$schema"`, значение которого равно `"https://raw.githubusercontent.com/Microsoft/Quantum/master/Chemistry/Schema/qchem-0.1.schema.json"`, и должно быть допустимым в соответствии со спецификацией JSON Schema-06 [[1](https://tools.ietf.org/html/draft-wright-json-schema-01), [2](https://tools.ietf.org/html/draft-wright-json-schema-validation-01)].

В оставшейся части этой спецификации объект Брумбридже будет ссылаться на объект JSON, десериализованный из документа YAML Брумбридже.

Если явно не указано иное, объекты не должны иметь дополнительных свойств, кроме указанных явно в этом документе.

## <a name="additional-definitions"></a>Дополнительные определения ##

Этот раздел является нормативным.

### <a name="quantity-objects"></a>Объекты Quantity ###

Этот раздел является нормативным.

_Объект Quantity_ является объектом JSON и должен иметь свойство `units`, значение которого является одним из допустимых значений, перечисленных в таблице 1.

Объект Quantity — это _простой объект Quantity_ , если в дополнение к свойству `units` имеется одно свойство `value`.
Значение свойства `value` должно быть числом.

Объект Quantity является _связанным объектом Quantity_ , если он содержит свойства `lower` и `upper` в дополнение к свойству `units`.
Значения свойств `lower` и `upper` должны быть числами.
Связанный объект Quantity может иметь свойство `value`, значением которого является число.

Объект Quantity — это _объект количества разреженных массивов_ , если он содержит свойство `format` и свойство `values` в дополнение к свойству `units`.
Значение `format` должно быть строкой `sparse`.
Значение свойства `values` должно быть массивом.
Каждый элемент `values` должен быть массивом, представляющим индексы, и значением количества разреженных массивов.

Индексы каждого элемента объекта количества разреженных массивов должны быть уникальными во всем объекте количества разреженных массивов.
Если индекс имеет значение `0`, то синтаксический анализатор должен обрабатывать объект Quantity разреженных массивов идентично объекту количества разреженных массивов, в котором этот индекс отсутствует.


Объект Quantity должен иметь значение

- Объект простого количества,
- связанный объект Quantity или
- Объект количества разреженных массивов.


### <a name="examples"></a>Примеры ###

Этот раздел является информативным.

Простое количество, представляющее $1.9844146837\,\Масрм{ха} $:

```yaml
coulomb_repulsion:
    value: 1.9844146837
    units: hartree
```

Ограниченное количество, представляющее равномерное распределение по интервалу $ [-7,-6]\,\Масрм{ха} $:

```yaml
fci_energy:
    upper: -6
    lower: -7
    units: hartree
```

Ниже приведено количество разреженных массивов с элементом, `[1, 2]` равным `hello`, а элемент `[3, 4]` равен `world`:

```yaml
sparse_example:
    format: sparse
    units: hartree
    values:
    - [1, 2, "hello"]
    - [3, 4, "world"]
```

## <a name="format-section"></a>Раздел формата ##

Этот раздел является нормативным.

Объект Брумбридже должен иметь свойство `format`, значение которого является объектом JSON с одним свойством, именуемым `version`.
Свойство `version` должно иметь значение `"0.1"`.

### <a name="example"></a>Пример ###

Этот раздел является информативным.

```yaml
format:                        # required
    version: "0.1"             # must match exactly
```

## <a name="integral-sets-section"></a>Раздел целых наборов ##

Этот раздел является нормативным.

Объект Брумбридже должен иметь свойство `integral_sets`, значение которого является массивом JSON.
Каждый элемент в значении свойства `integral_sets` должен быть объектом JSON, описывающим один набор целых чисел, как описано в оставшейся части этого раздела.
В оставшейся части этого раздела термин «объект целого набора» будет ссылаться на элемент в значении свойства `integral_sets` объекта Брумбридже.

Каждый объект целочисленного набора должен иметь свойство `metadata`, значение которого является объектом JSON.
Значение `metadata` может быть пустым объектом JSON (то есть `{}`) или может содержать дополнительные свойства, определенные разработчиком.

### <a name="hamiltonian-section"></a>Раздел хамилтониан ###

#### <a name="overview"></a>Краткое описание ####

Этот раздел является информативным.

Свойство `hamiltonian` каждого объекта целочисленного набора описывает Хамилтониан для определенной проблемы такта химия, вычисляя его один-и два основных термина в виде разреженных массивов вещественных чисел.
Операторы Хамилтониан, описанные каждым объектом целочисленного набора, принимают форму

$ $ H = \сум\_\{i, j\}\сум\_{\сигма\ин\\{\упарров, \довнарров\\}} H\_\{ИЖ\} а ^\{\дагжер\}\_{i , \сигма}\_{j, \сигма} + \фрак{1}{2}\сум\_\{i, j, k, l\}\сум\_{\сигма, \рхо\ин\\{\упарров, \довнарров\\}} h\_{ижкл} a ^ \дагжер\_{i , \сигма} a ^ \дагжер\_{k, \рхо}\_{l, \рхо}\_{j, \сигма}, $ $

Здесь $h _ {ижкл} = (ИЖ | КЛ) $ в соглашении Мулликен.

Для ясности термина «один электронное» —

$ $ H_ {ИЖ} = \инт {\масрм d} x \пси ^ *\_i (x) \лефт (\фрак{1}{2}\набла ^ 2 + \сум\_{A} \Фрак{з\_а} {| x-x\_A |}  \ригхт) \пси\_j (x), $ $

и два электронного выражения:

$ $ h\_\{ижкл\} = \иинт \{\масрм d\}x ^ 2 \пси ^\{\*\}\_i (x\_1) \пси\_j (x\_1) \фрак\{1\}\{\|x\_1-x\_2\|\}\пси\_k ^\{\*\}(x\_2) \пси\_l (x\_2).
$$

Как отмечалось в описании [свойства`basis_set`](#basis-set-object) каждого элемента свойства `integral_sets`, мы еще дальше предполагаю, что используемые основные функции являются реальными.
Это позволяет нам использовать следующие симметриес между терминами для сжатия представления Хамилтониан.

$ $ H_ {ижкл} = H_ {ижлк} = H_ {жикл} = H_ {жилк} = H_ {клиж} = H_ {клжи} = H_ {лкиж} = H_ {лкжи}.
$$


#### <a name="contents"></a>Контент ####

Этот раздел является нормативным.

Каждый целый набор должен иметь свойство `hamiltonian`, значение которого является объектом JSON.
Значение свойства `hamiltonian` известно как объект Хамилтониан и должно иметь свойства `one_electron_integrals` и `two_electron_integrals`, как описано в оставшейся части этого раздела.
Объект Хамилтониан также может иметь свойство `particle_hole_representation`.
При наличии значение `particle_hole_representation` должно соответствовать формату, описанному в оставшейся части этого раздела.

##### <a name="one-electron-integrals-object"></a>Объект «интегралы за один электронный» #####

Этот раздел является нормативным.

Свойство `one_electron_integrals` объекта Хамилтониан должно быть разреженным массивом, индексом которого является два целых числа и значения которых являются числами.
Каждый термин должен иметь индексы `[i, j]`, где `i >= j`.

> МЕТИМ Это отражает симметрию, которая $h _ {ИЖ} = H_ {ji} $, что является следствием того факта, что Хамилтониан является Хермитиан.


###### <a name="example"></a>Пример ######

Этот раздел является информативным.

Следующее количество разреженных массивов представляет Хамилтониан $ $ H = \лефт (-5,0 (a ^\{\дагжер\}\_{1, \упарров}\_{1, \упарров} + a ^\{\дагжер\}\_{1, \довнарров} a\_{1 , \довнарров}) + 0,17 (a ^\{\дагжер\}\_{2, \упарров} a\_{1, \упарров} + a ^\{\дагжер\}\_{1, \упарров} a\_{2, \упарров} + a ^\{\дагжер\}\_{2 , \довнарров}\_{1, \довнарров} + a ^\{\дагжер\}\_{1, \довнарров} a\_{2, \довнарров}) \ригхт)\\, \Масрм{ха}.
$$

```yaml
one_electron_integrals:     # required
    units: hartree          # required
    format: sparse          # required
    values:                 # required
        # i j f(i,j)
        - [1, 1, -5.0]
        - [2, 1,  0.17]
```
> [!NOTE]
> Брумбридже использует индексирование, основанное на 1.


##### <a name="two-electron-integrals-object"></a>Объект интегралов с двумя электронными данными #####

Этот раздел является нормативным.

Свойство `two_electron_integrals` объекта Хамилтониан должно быть количеством разреженных массивов с одним дополнительным свойством с именем `index_convention`.
Каждый элемент значения `two_electron_integrals` должен иметь четыре индекса.

Каждое свойство `two_electron_integrals` должно иметь свойство `index_convention`.
Значение свойства `index_convention` должно быть одним из допустимых значений, перечисленных в таблице 1.
Если значение `index_convention` равно `mulliken`, то для каждого элемента `two_electron_integrals` количество разреженных массивов средство синтаксического анализа, загружая документ Брумбридже, должно создать экземпляр Хамилтониан, равный оператору Two-электронного оператора $h _ {i, j, k, l} а ^ \dagger_i a ^ \dagger_j a_k a_l $ , где $i $, $j $, $k $ и $l $ должны быть целыми числами в диапазоне от 1 до количества электронов, заданных свойством `n_electrons` объекта целочисленного набора, и где $h _ {i, j, k, l} $ является элементом, `[i, j, k, l, h(i, j, k, l)]`ным для количества разреженных массивов.

###### <a name="symmetries"></a>симметриес ######

Этот раздел является нормативным.

Если свойство `index_convention` объекта `two_electron_integrals` равно `mulliken`, то при наличии элемента с индексами `[i, j, k, l]` следующие индексы не должны присутствовать, если они не равны `[i, j, k, l]`:

- `[i, j, l, k]`
- `[j, i, k, l]`
- `[j, i, l, k]`
- `[k, l, i, j]`
- `[k, l, j, i]`
- `[l, k, j, i]`

> [!NOTE]
> Поскольку свойство `index_convention` является объектом разреженного количества, индексы не могут повторяться в разных элементах.
> В частности, если имеется элемент с индексами `[i, j, k, l]`, другие элементы не могут содержать эти индексы.

###### <a name="example"></a>Пример #######

Этот раздел является информативным.

Следующий объект указывает Хамилтониан

##### <a name="particlehole-representation-object"></a>Частица — объект представления отверстий #####

Этот раздел является нормативным.

Объект представления частица (дыра) указывает, что хранимые интегралы определяются по отношению к части частицы, где операторы создания и аннихилатион описывают ексЦитатионс от используемого состояния ссылки, например Хартри — Состояние Фокк.
Объект является необязательным.
Если объект не указан, Хамилтониан интерпретируется как не заданный в представлении частиц.
Если этот параметр задан, значение `particle_hole_representation` должно быть разреженным массивом объектов Quantity, индексами которого являются четыре целых числа, а значения — число и строка.
Строковая часть значения каждого элемента должна содержать только символы `'+'` и `'-'`, указывающие, является ли данный фактор в термине оператором создания или аннихилатион в представлении частица — отверстия.  Например `"-+++"` взаимореакций на отверстие, создаваемое на сайтах $i $ и частицы, создаваемые на узлах $j, k $ и $l $.

> [!NOTE]
> Так как значение `particle_hole_representation` является объектом количества разреженных массивов, необходимо указать свойства `unit` и `format`.
> Допустимые единицы перечислены в таблице 1.
> Свойство `format` является обязательным и указывает, указываются ли коэффициенты Хамилтониан в виде сжатого или разреженного массива.
> В текущей версии поддерживаются только разреженные массивы с интерпретацией того, что все неопределенные элементы имеют $0 $, но в будущих версиях может быть добавлена поддержка дополнительных значений свойства `format`.

### <a name="initial-state-section"></a>Раздел начального состояния ###

Объект initial_state_suggestion Указывает начальные состояния такта, представляющие интерес для указанного Хамилтониан. Этот объект должен быть массивом объектов JSON `state`.

#### <a name="state-object"></a>Объект State ####

Каждое из состояний представляет собой перестановку занятых орбит. Каждый объект состояния должен иметь свойство `label`, содержащее строку. Каждый объект состояния должен иметь свойство `superposition`, содержащее массив состояний и их ненормализованные амплитуды.

Например, начальное состояние $ $ \ket{G0} = \ket{G1} = \ket{G2} = (a ^ {\дагжер}\_{1, \упарров}а ^ {\дагжер}\_{2, \упарров}а ^ {\дагжер}\_{2, \довнарров}) \кет{0} $ $ $ $ \Кет{е} = \frac{0.1 (a ^ {\дагжер}\_{1 , \упарров}а ^ {\дагжер}\_{2, \упарров}а ^ {\дагжер}\_{2, \довнарров}) + 0,2 (a ^ {\дагжер}\_{1, \упарров}а ^ {\дагжер}\_{3, \упарров}а ^ {\dagger}\_{2, \downarrow})} {\sqrt{| 0,1 | ^ 2 + | 0,2 | ^ 2}} \ket{0} $ $ представлено
```yaml
initial_state_suggestions: # optional. If not provided, spin-orbitals will be filled to minimize one-body diagonal term energies.
    - state:
        label: "|G0>"
        superposition:
            - [1.0, "(1a)+","(2a)+","(2b)+","|vacuum>"]
    - state:
        label: "|G1>"
        superposition:
            - [-1.0, "(2a)+","(1a)+","(2b)+","|vacuum>"]
    - state:
        label: "|G2>"
        superposition:
            - [1.0, "(3a)","(1a)+","(2a)+","(3a)+","(2b)+","|vacuum>"]
    - state:
        label: "|E>"
        superposition:
            - [0.1, "(1a)+","(2a)+","(2b)+","|vacuum>"]
            - [0.2, "(1a)+","(3a)+","(2b)+","|vacuum>"]
```

#### <a name="basis-set-object"></a>Объект базового набора ####

Этот раздел является нормативным.

Каждый объект целочисленного набора может иметь свойство `basis_set`.
При наличии значение свойства `basis_set` должно быть объектом с двумя свойствами: `type` и `name`.

Основные функции, определяемые значением свойства `basis_set`, должны быть реальными.

> [!NOTE]
> Предположение, что все основные функции являются реальными, могут быть неослабленными в будущих версиях этой спецификации.

## <a name="tables-and-lists"></a>Таблицы и списки ##

### <a name="table-1-allowed-physical-units"></a>Таблица 1. Разрешенные физические устройства ###

Этот раздел является нормативным.

Любая строка, задающая единицу, должна быть одной из следующих:

- `hartree`
- `ev`

Средства синтаксического анализа и производители должны рассматривать следующие простые объекты Quantity как эквивалентные:

```yaml
- {"units": "hartree", "value": 1}
- {"units": "ev", "value": 27.2113831301723}
```

### <a name="table-2-allowed-index-conventions"></a>Таблица 2. Допустимые соглашения об индексах ###

Этот раздел является нормативным.

Любая строка, задающая соглашение об индексах, должна быть одной из следующих:
