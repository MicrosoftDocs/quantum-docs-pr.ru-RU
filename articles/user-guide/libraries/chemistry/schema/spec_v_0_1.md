# <a name="broombridge-specification-v01"></a>Спецификация брумбридже 0,1 #

Ключевые слова "обязательно", "не должны", "обязательно", "должно", "не", "должно", "не должно", "рекомендуется", "Май" и "необязательно" в этом документе должны интерпретироваться, как описано в [RFC 2119](https://tools.ietf.org/html/rfc2119).

Любая боковая панель с заголовками «Примечание», «информация» или «предупреждение» является информативной.

## <a name="introduction"></a>Введение ##

Этот раздел является информативным.

Документы брумбридже предназначены для обмена экземплярами проблем моделирования в тактовой химия для обработки с помощью моделирования тактов и программирования цепочек инструментов.

## <a name="serialization"></a>Сериализация ##

Этот раздел является нормативным.

Документ Брумбридже должен быть сериализован как [документ YAML 1,2](http://yaml.org/spec/) , представляющий объект JSON, как описано в [RFC 4627](https://tools.ietf.org/html/rfc4627) , раздел 2,2.
Объект, сериализованный в YAML, должен иметь свойство `"$schema"` , значение которого равно `"https://raw.githubusercontent.com/Microsoft/Quantum/master/Chemistry/Schema/qchem-0.1.schema.json"` , и должно быть допустимым в соответствии с требованиями схемы JSON — 06 [[1](https://tools.ietf.org/html/draft-wright-json-schema-01), [2](https://tools.ietf.org/html/draft-wright-json-schema-validation-01)].

В оставшейся части этой спецификации объект Брумбридже будет ссылаться на объект JSON, десериализованный из документа YAML Брумбридже.

Если явно не указано иное, объекты не должны иметь дополнительных свойств, кроме указанных явно в этом документе.

## <a name="additional-definitions"></a>Дополнительные определения ##

Этот раздел является нормативным.

### <a name="quantity-objects"></a>Объекты Quantity ###

Этот раздел является нормативным.

_Объект Quantity_ является объектом JSON и должен иметь свойство `units` , значение которого является одним из допустимых значений, перечисленных в таблице 1.

Объект Quantity является _простым объектом Quantity_ , если `value` в дополнение к его свойству имеется единственное свойство `units` .
Значение `value` свойства должно быть числом.

Объект Quantity — это _привязанный объект Quantity_ , если он содержит свойства `lower` и `upper` в дополнение к его `units` свойству.
Значения `lower` `upper` свойств и должны быть числами.
Связанный объект Quantity может иметь свойство `value` , значение которого является числом.

Объект Quantity — это _объект количества разреженных массивов_ , если он содержит свойство `format` и свойство `values` в дополнение к его `units` свойству.
Значение `format` должно быть строкой `sparse` .
Значение `values` свойства должно быть массивом.
Каждый элемент `values` должен быть массивом, представляющим индексы, и значением количества разреженных массивов.

Индексы каждого элемента объекта количества разреженных массивов должны быть уникальными во всем объекте количества разреженных массивов.
Если индекс имеет значение `0` , синтаксический анализатор должен обрабатывать объект Quantity разреженного массива идентично объекту Quantity разреженного массива, в котором этот индекс отсутствует.


Объект Quantity должен иметь значение

- Объект простого количества,
- связанный объект Quantity или
- Объект количества разреженных массивов.


### <a name="examples"></a>Примеры ###

Этот раздел является информативным.

Простое количество, представляющее $1.9844146837 \, \масрм{ха} $:

```yaml
coulomb_repulsion:
    value: 1.9844146837
    units: hartree
```

Ограниченное количество, представляющее равномерное распределение по интервалу $ [-7,-6] \, \масрм{ха} $:

```yaml

    upper: -6
    lower: -7
    units: hartree
```

Ниже приведено количество разреженных массивов с элементом `[1, 2]` , равным `hello` , и элементом, `[3, 4]` равным `world` :

```yaml
sparse_example:
    format: sparse
    units: hartree
    values:
    - [1, 2, "hello"]
    - [3, 4, "world"]
```

## <a name="integral-sets-section"></a>Раздел целых наборов ##

Этот раздел является нормативным.

Объект Брумбридже должен иметь свойство `integral_sets` , значение которого является массивом JSON.
Каждый элемент в значении `integral_sets` свойства должен быть объектом JSON, описывающим один набор целых чисел, как описано в оставшейся части этого раздела.
В оставшейся части этого раздела термин «объект целого набора» будет ссылаться на элемент в значении `integral_sets` свойства объекта брумбридже.

Значение `metadata` может быть пустым объектом JSON (то есть `{}` ) или может содержать дополнительные свойства, определенные разработчиком.

### <a name="hamiltonian-section"></a>Раздел хамилтониан ###

#### <a name="overview"></a>Обзор ####

Этот раздел является информативным.

`hamiltonian`Свойство каждого объекта целочисленного набора описывает хамилтониан для определенной проблемы тактовой химия, вычисляя его один-и два основных термина в виде разреженных массивов вещественных чисел.
Операторы Хамилтониан, описанные каждым объектом целочисленного набора, принимают форму

$ $ H = \сум \_ \{ i, j \} \сум \_ {\сигма\ин \\ {\упарров, \довнарров \\ }} H \_ \{ ИЖ \} a ^ \{ \дагжер \} \_ {i, \сигма}. \_ {j, \сигма} + \фрак {1} {2} \сум \_ \{ i, j, k, l \} \сум \_ {\сигма, \rho\in \\ {\uparrow, \downarrow \\ }} H \_ {ijkl} a ^ \dagger \_ {i, \sigma} a ^ \dagger \_ {k, \rho} a \_ {l, \rho} a \_ {j, \sigma}, $ $

Здесь $h _ {ижкл} = (ИЖ | КЛ) $ в соглашении Мулликен.

Для ясности термина «один электронное» —

$ $ h_ {ИЖ} = \инт {\масрм d} x \пси ^ * \_ i (x) \лефт (\фрак {1} {2} \набла ^ 2 + \сум \_ {A} \фрак{з \_ а} {| x-x \_ A |}  \ригхт) \пси \_ j (x), $ $

и два электронного выражения:

$ $ h \_ \{ ижкл \} = \иинт \{ \масрм d \} x ^ 2 \пси ^ \{ \* \} \_ i (x \_ 1) \пси \_ j (x \_ 1) \фрак \{ 1 \} \{ \| x \_ 1-x \_ 2 \| \} \пси \_ k ^ \{ \* \} (x \_ 2) \пси \_ l (x \_ 2).
$$

Как отмечалось в описании [ `basis_set` свойства](#basis-set-object) каждого элемента `integral_sets` свойства, мы еще дальше предполагаю, что используемые основные функции являются реальными.
Это позволяет нам использовать следующие симметриес между терминами для сжатия представления Хамилтониан.

$ $ h_ {ижкл} = h_ {ижлк} = h_ {жикл} = h_ {жилк} = h_ {клиж} = h_ {клжи} = h_ {лкиж} = h_ {лкжи}.
$$


#### <a name="contents"></a>Содержимое ####

Этот раздел является нормативным.

Каждый целый набор должен иметь свойство `hamiltonian` , значение которого является объектом JSON.
Значение `hamiltonian` Свойства известно как объект хамилтониан и должно иметь свойства `one_electron_integrals` и `two_electron_integrals` , как описано в оставшейся части этого раздела.
Объект Хамилтониан также может иметь свойство `particle_hole_representation` .
При наличии значение `particle_hole_representation` должно соответствовать формату, описанному в оставшейся части этого раздела.

##### <a name="one-electron-integrals-object"></a>One-Electronный объект интегралов #####

Этот раздел является нормативным.

`one_electron_integrals`Свойство объекта ХАМИЛТОНИАН должно быть разреженным массивом, индексом которого является два целых числа и значения которых являются числами.
Каждый термин должен иметь индексы `[i, j]` `i >= j` , где.

> МЕТИМ Это отражает симметрию, которая $h _ {ИЖ} = h_ {ji} $, что является следствием того факта, что Хамилтониан является Хермитиан.

Этот раздел является информативным.

Следующее количество разреженных массивов представляет хамилтониан $ $ H = \лефт (-5,0 (a ^ \{ \дагжер \} \_ {1, \упарров} a \_ {1, \упарров} + a ^ \{ \дагжер \} \_ {1, \довнарров} \_ {1, \довнарров}) + 0,17 (a ^ \{ \дагжер \} \_ {2, \uparrow} a \_ {1, \uparrow} + a ^ \{ \dagger \} \_ {1, \uparrow}. \_ {2, \uparrow} + a ^ \{ \dagger {2, \downarrow} a { \} \_ \_ 1, \downarrow} + a ^ \dagger \{ \} \_ {1, \downarrow} a \_ {2, \downarrow}) \right) \\ , \mathrm{ha}.
$$

```yaml
one_electron_integrals:     # required
    units: hartree          # required
    format: sparse          # required
    values:                 # required
        # i j f(i,j)
```

##### <a name="two-electron-integrals-object"></a>Two-Electronный объект интегралов #####

Этот раздел является нормативным.

`two_electron_integrals`Свойство объекта ХАМИЛТОНИАН должно быть количеством разреженных массивов с одним дополнительным свойством с именем `index_convention` .
Каждый элемент значения `two_electron_integrals` должен иметь четыре индекса.

Каждое `two_electron_integrals` свойство должно иметь `index_convention` свойство.
Значение `index_convention` свойства должно быть одним из допустимых значений, перечисленных в таблице 1.
Если значение `index_convention` равно `mulliken` , то для каждого элемента с `two_electron_integrals` количеством разреженных массивов средство синтаксического анализа, загружая документ брумбридже, должно создать экземпляр хамилтониан, равный двум электронным операторам $h _ {i, j, k, l} a ^ \ dagger_i a ^ \ dagger_j a_k a_l $, где $i $, $j $, $k $ и $l $ должны быть целыми числами в диапазоне от 1 до числа электронов, заданных `n_electrons` свойством объекта целочисленного набора, и где $h _ {i, j, k, l} $ является элементом `[i, j, k, l, h(i, j, k, l)]` количества разреженного массива.

###### <a name="symmetries"></a>симметриес ######

Этот раздел является нормативным.

Если `index_convention` свойство `two_electron_integrals` объекта равно `mulliken` , то при наличии элемента с индексами `[i, j, k, l]` следующие индексы не должны присутствовать, если они не равны `[i, j, k, l]` :

- `[i, j, l, k]`
- `[j, i, k, l]`
- `[j, i, l, k]`
- `[k, l, i, j]`
- `[k, l, j, i]`
- `[l, k, j, i]`

> [!NOTE]
> Поскольку `index_convention` свойство является объектом разреженного количества, индексы не могут повторяться в разных элементах.
> В частности, если имеется элемент с индексами `[i, j, k, l]` , другие элементы не могут иметь эти индексы.

##### <a name="particlehole-representation-object"></a>Частица — объект представления отверстий #####

Этот раздел является нормативным.

Объект представления частица (дыра) указывает, что хранимые интегралы определяются по отношению к части частицы, где операторы создания и аннихилатион описывают ексЦитатионс от используемого состояния ссылки, например Хартри – Фокк State.
Объект является необязательным.
Если объект не указан, Хамилтониан интерпретируется как не заданный в представлении частиц.
При наличии значение `particle_hole_representation` должно быть разреженным массивом объектов Quantity, индексами которого являются четыре целых числа, а значения — числом и строкой.
Строковая часть значения каждого элемента должна содержать только символы `'+'` и `'-'` , указывающие, является ли данный фактор в термине оператором создания или аннихилатион в представлении частица — отверстие.  Например, `"-+++"` обреагирует на отверстие, создаваемое на сайте $i $ и частицы, создаваемые на сайтах $j, k $ и $l $.

> [!NOTE]
> Так как значение `particle_hole_representation` является объектом количества разреженных массивов, `unit` `format` необходимо указать свойства и.
> Допустимые единицы перечислены в таблице 1.
> `format`Свойство является обязательным и указывает, указаны ли коэффициенты хамилтониан в виде сжатого или разреженного массива.
> В текущей версии поддерживаются только разреженные массивы с интерпретацией того, что все неопределенные элементы имеют $0 $, но в будущих версиях может быть добавлена поддержка дополнительных значений `format` Свойства.

### <a name="initial-state-section"></a>Раздел начального состояния ###

Объект initial_state_suggestion Указывает начальные состояния такта, представляющие интерес для указанного Хамилтониан. Этот объект должен быть массивом объектов JSON `state` .

#### <a name="state-object"></a>Объект State ####

Каждое из состояний представляет собой перестановку занятых орбит. Каждый объект состояния должен иметь `label` свойство, содержащее строку. Каждый объект состояния должен иметь `superposition` свойство, содержащее массив состояний и их ненормализованные амплитуды.

Например, начальное состояние $ $ \ket{G0} = \ket{G1} = \ket{G2} = (a ^ {\дагжер} \_ {1, \упарров}а ^ {\дагжер} \_ {2, \упарров}а ^ {\дагжер} \_ {2, \довнарров}) \кет {0} $ $ $ $ \кет{е} = \frac{0.1 (a ^ {\дагжер} \_ {1, \упарров}а ^ {\dagger} \_ {2, \uparrow}a ^ {\dagger} \_ {2, \downarrow}) + 0,2 (a ^ {\dagger} \_ {1, \uparrow}a ^ {\dagger} \_ {3, \uparrow}a ^ {\dagger} \_ {2, \downarrow})} {\sqrt{| 0,1 | ^ 2 + | 0,2 | ^ 2}} \ket {0} $ $ представлены
```yaml
initial_state_suggestions: # optional. If not provided, spin-orbitals will be filled to minimize one-body diagonal term energies.
    - state:
        label: "|G0>"
        superposition:
            - [1.0, "(1a)+","(2a)+","(2b)+","|vacuum>"]
    - state:
        label: "|G1>"
        superposition:
            - [-1.0, "(2a)+","(1a)+","(2b)+","|vacuum>"]
    - state:
        label: "|G2>"
        superposition:
            - [1.0, "(3a)","(1a)+","(2a)+","(3a)+","(2b)+","|vacuum>"]
    - state:
        label: "|E>"
        superposition:
            - [0.1, "(1a)+","(2a)+","(2b)+","|vacuum>"]
            - [0.2, "(1a)+","(3a)+","(2b)+","|vacuum>"]
```

#### <a name="basis-set-object"></a>Объект базового набора ####

Этот раздел является нормативным.

Каждый объект целочисленного набора может иметь `basis_set` свойство.
При наличии значение `basis_set` свойства должно быть объектом с двумя свойствами `type` и `name` .

Основные функции, определяемые значением `basis_set` свойства, должны быть реальными.

> [!NOTE]
> Предположение, что все основные функции являются реальными, могут быть неослабленными в будущих версиях этой спецификации.
